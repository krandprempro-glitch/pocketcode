## 功能1: Tab1 快捷指令添加收藏文件夹 CD 切换

### 需求描述
在终端页面（Tab1）的快捷指令菜单中，添加收藏路径分组，允许用户快速跳转到收藏的文件夹。

### 设计方案

#### 菜单结构
```
┌─────────────────────────────────────┐
│  快捷指令                    [X]    │
├─────────────────────────────────────┤
│ ▶ 📁 收藏路径 (默认折叠)             │
│   📂 /home/user/projects      →     │
│   📂 /var/log                 →     │
│   📂 /etc/nginx               →     │
├─────────────────────────────────────┤
│ ▶ 🔌 SSH连接 (默认折叠)              │
├─────────────────────────────────────┤
│ ▶ 🤖 AI指令 (默认折叠)               │
├─────────────────────────────────────┤
│ ▶ ⚙️ 系统指令 (默认折叠)             │
└─────────────────────────────────────┘
```

#### 实现细节
- **数据来源**: 复用 `BookmarkRepository` 获取当前连接的收藏列表
- **命令格式**: `cd <bookmark.path>`
- **交互方式**: 点击收藏项直接发送命令到终端
- **默认状态**: 所有分组默认折叠，减少菜单高度
- **空状态**: 无收藏时显示"暂无收藏路径"

#### 代码位置
- 修改: `TermuxFragment.java` - `prepareCommandGroups()` 方法
- 新增: 从 `BookmarkRepository` 获取收藏数据
- 新增: 收藏路径命令分组

## 功能2: 文件浏览器顶部状态栏简化

### 需求描述
简化顶部 Toolbar 显示，将"访问目录+ip+已连接"改为目录名+绿点表示连接状态。

### 设计方案

#### 修改前
```
┌─────────────────────────────────────┐
│  ≡  /home/user/project  🔴 未连接    │
│       ↑    ↑    ↑    ↑    ↑         │
│    菜单  完整路径    图标  文字      │
└─────────────────────────────────────┘
```

#### 修改后
```
┌─────────────────────────────────────┐
│  ≡  project  ●                      │
│       ↑    ↑    ↑                   │
│    菜单  目录名 绿点(已连接)          │
│            或  ○ 灰点(未连接)        │
└─────────────────────────────────────┘
```

#### 实现细节
- **目录名显示**: 提取当前路径的最后一段（如 `/home/user/project` → `project`）
- **连接状态**:
  - 已连接: 绿色圆点 ● (Android 绿色 `#4CAF50`)
  - 未连接: 灰色圆点 ○ (Android 灰色 `#9E9E9E`)
- **长按提示**: 长按状态点显示完整连接信息（IP地址）
- **代码位置**: `fragment_remote_file_browser_drawer.xml` Toolbar 区域

## 功能3: 文件浏览器双栏布局重构

### 需求描述
将左侧抽屉改为 Tab2 主体区域的双栏布局，左侧显示文件目录树，右侧显示文件内容。

### 设计方案

#### 布局结构
```
┌─────────────────────────────────────────────────────┐
│  Toolbar: ≡ 📁 project-name  ●                      │
├─────────────────────────────────────────────────────┤
│  📂 文件树 (30%)    │  主内容区 (70%)                │
│                     │                               │
│  📁 src             │  ┌─────────────────────────┐  │
│  ├── 📄 main.java   │  │ 文件夹视图               │  │
│  │   └── 📄 App.kt  │  │                         │  │
│  ├── 📁 res         │  │ 📁 assets    📁 src      │  │
│  └── 📄 README.md   │  │ 📁 config    📄 build... │  │
│                     │  │                         │  │
│  [可拖拽分割线]      │  └─────────────────────────┘  │
├─────────────────────┼───────────────────────────────┤
│ 底部状态栏          │  [文件列表/网格视图]            │
└─────────────────────┴───────────────────────────────┘
```

#### 状态1: 文件夹浏览模式
- 右侧主区域显示当前目录的文件/文件夹网格
- 点击文件夹进入子目录
- 点击代码/文本文件进入编辑器

#### 状态2: 代码编辑器模式
```
┌─────────────────────────────────────────────────────┐
│  ←  📄 main.java  [保存]  [关闭]                    │
├─────────────────────────────────────────────────────┤
│  1 │ package com.example;                           │
│  2 │                                                │
│  3 │ public class Main {                            │
│  4 │     public static void main(String[] args) {   │
│  5 │         System.out.println("Hello");           │
│  6 │     }                                          │
│  7 │ }                                              │
│     │                                                │
└─────────────────────────────────────────────────────┘
```

#### 实现细节
- **布局组件**:
  - 使用 `LinearLayout` 水平分割（权重 30/70）
  - 中间添加 `View` 作为可拖拽分割线
- **导航逻辑**:
  - 左侧文件树点击文件夹 → 右侧显示该文件夹内容
  - 右侧点击文件夹 → 进入该文件夹
  - 右侧点击代码文件 → 打开编辑器（带返回按钮）
- **返回逻辑**:
  - 编辑器返回按钮 → 回到文件夹浏览模式
  - 系统返回键 → 优先返回上一级目录，到根目录时退出

#### 代码位置
- 修改: `fragment_remote_file_browser_drawer.xml`
- 修改: `RemoteFileBrowserFragment.kt`
- 新增: 文件夹网格适配器
- 新增: 编辑器返回处理

## 技术实现要点

### 依赖注入
- 使用已有的 `FileBrowserModule` 注入 `BookmarkRepository`

### 数据流
1. **快捷指令菜单**:
   ```
   TermuxFragment -> BookmarkRepository -> BookmarkDataSource -> SharedPreferences
   ```

2. **文件浏览器**:
   ```
   文件树点击 -> ViewModel -> NavigateToDirectoryUseCase -> SftpRepository
   ```

### 兼容性
- 保持与现有 SSH 连接逻辑兼容
- 保持与现有收藏功能兼容
